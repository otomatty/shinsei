### UI/UX関連

#### iframe埋め込み時のファイル名表示機能

iframe内でLichtblickが埋め込まれている場合、リモートファイルのURL全体ではなくファイル名のみを表示する機能。

**📚 ドキュメント:**

- **[アプローチ比較](./iframe-embedded-approaches-comparison.md)** ⭐ まずはこちら

  - 3つの実装アプローチの比較と推奨
  - 定量的・定性的な比較表
  - 選択基準と決定マトリックス

- **[iframe判定分離アプローチ](./iframe-embedded-filename-display-separated-approach.md)** ⭐⭐⭐ 推奨

  - iframe検出を独立ファイルに分離（再利用可能）
  - 2ファイル（約80行）で実装
  - 実装時間: 約1時間
  - バランスの取れた設計

- **[完全統合アプローチ](./iframe-embedded-filename-display-minimal-approach.md)**

  - 既存コードへの変更を最小限に抑えた実装方法
  - 1ファイル（約50行）で完結
  - 実装時間: 約35分
  - 最もシンプルだが再利用性は低い

- **[完全分離アプローチ](./iframe-embedded-filename-display.md)**

  - 保守性・再利用性を最大化した実装方法
  - 4ファイル（約410行）に完全分離
  - 実装時間: 約2時間
  - 大規模プロジェクト向け

- **[クイックリファレンス](./iframe-embedded-filename-display-quick-reference.md)**
  - 実装の要点を簡潔にまとめたリファレンス

**🎯 推奨**: iframe判定分離アプローチ（アプローチB）を採用

- iframe検出の再利用性: ⭐⭐⭐⭐
- 実装コスト: 中程度（約1時間）
- 保守性: ⭐⭐⭐⭐
- 総合評価: 37/40点

## 📚 ドキュメントの使い方

### 新規機能を実装する場合

1. **計画フェーズ**: 既存の実装ドキュメントを参照し、類似機能の実装パターンを確認
2. **設計フェーズ**: 必要に応じて新しい実装ドキュメントを作成
3. **実装フェーズ**: ドキュメントに従って実装を進める
4. **レビューフェーズ**: ドキュメントと実装の整合性を確認

### 実装ドキュメントのテンプレート

新しい実装ドキュメントを作成する際は、以下の構成を推奨します：

```markdown
# [機能名] 実装ドキュメント

## 📋 概要

- 目的
- 実装アプローチ
- 影響範囲

## 🏗️ アーキテクチャ概要

- システム構成図
- データフロー

## 📁 実装ファイル一覧

### 新規作成ファイル

### 修正対象ファイル

### 関連する既存ファイル

## 🔧 詳細実装仕様

- 各ファイルの実装内容
- コードサンプル

## 🧪 テスト実装

- ユニットテスト
- 統合テスト
- E2Eテスト

## 📊 データフロー詳細

- 初期化フロー
- 実行フロー

## 🔍 既存ファイルとの関連性

- 影響を受けるファイル
- 参考にすべきファイル

## 🎯 実装手順

- Phase 1, 2, 3...

## 🧪 テスト実行方法

- テストコマンド
- 動作確認方法

## 🐛 トラブルシューティング

- よくある問題と解決策

## 📝 コミットメッセージ例

## 🚀 今後の拡張可能性

## ✅ チェックリスト

## 📚 参考資料
```

## 📌 命名規則

実装ドキュメントのファイル名は以下の規則に従います：

- **機能名-実装内容.md**: `iframe-embedded-filename-display.md`
- **コンポーネント名-機能名.md**: `desktop-extension-loader-improvement.md`
- **問題名-fix.md**: `minimal-fix-for-marketplace-persistence.md`

すべて小文字とハイフン（-）を使用します。
