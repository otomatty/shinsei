- チーム内での一貫した作業環境が生産性に直結する
- 現在のレイアウト作成は「属人的スキル」に依存している

#### 1. 発見フェーズ（Discovery Phase）

```
ユーザーの行動パターン:
├─ 目的から検索: "自動運転 監視" "ドローン 制御"
├─ カテゴリーから探索: "Automotive" → "Autonomous Driving"

#### 2. 評価フェーズ（Evaluation Phase）

```
レイアウト詳細画面での判断要素:
├─ スクリーンショット: 一目で分かる機能確認
├─ 使用シーン説明: 自分の用途とのマッチング

#### 3. 試用フェーズ（Trial Phase）

```

├─ ワンクリックインストール: 即座に利用開始
├─ サンプルデータ提供: 動作確認の簡易化

```

├─ チーム内での共有・評価

## 🔍 現在のレイアウト利用シーンと課題分析

### 利用パターン1: プロジェクト開始時のセットアップ

#### 現在の典型的なワークフロー

```
📅 Day 1-3: レイアウト設計フェーズ
├─ 既存プロジェクトのレイアウトを参考にする
├─ 必要なパネルを手動で配置
├─ トピック名やパラメータを試行錯誤で調整
└─ チームメンバーが各自で同様の作業を繰り返す

⏰ 総所要時間: 1人あたり2-4時間
💸 人件費: 約10,000-20,000円/人
😫 フラストレーション: 高（創造的でない作業に時間を取られる）
```

#### 具体的な課題

**1. パネル配置の試行錯誤**

```
問題:
- どのパネルをどこに配置すべきか分からない
- パネルサイズの最適化に時間がかかる
- 3Dビューと2Dビューのバランス調整が困難

実際の声:
「新しいプロジェクトの度に、レイアウトを一から作り直すのが面倒」
「どのパネルが自動運転監視に最適か判断できない」
```

**2. トピック名の不一致**

```
問題:
- プロジェクトごとにトピック名が異なる
- グローバル変数の設定ミス
- パネル設定の不整合

実際の声:
「同僚のレイアウトをコピーしても、トピック名が違って動かない」
「変数の設定方法が分からず、結局手動で全部修正している」
```

```

⏰ レイアウト調整時間: 1日30分-1時間
📊 効率性: 60%（レイアウト切り替えのオーバーヘッド）
```

#### 具体的な課題

**1. 作業コンテキストの切り替えコスト**

```

├─ 3Dマップビュー中心のレイアウト

└─ 計画経路との比較

├─ LiDARポイントクラウド表示
├─ カメラ画像とLiDARの重畳表示

問題:
- レイアウトの手動切り替えに5-10分
- パネル配置を思い出すのに時間がかかる
- 設定値を毎回調整する必要
```

**2. チーム内での情報共有困難**

```

問題:
- 同じ「見え方」を共有できない
- レイアウトの口頭説明は非効率
- バグの再現に時間がかかる
```
